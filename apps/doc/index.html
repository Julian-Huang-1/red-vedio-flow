Local-first agent orchestration

让本地 Agent CLI，真正参与视频生产。

Red Video Flow 把你已经在用的 Codex、Claude Code、Gemini CLI 等本地 Agent，接进脚本、分镜、图片和视频节点。上下文自动传递，结果自动回写，创作过程清晰可追踪。

1条从灵感到成片的连续工作流

不是另一个聊天框，
而是可执行的创作节点。

Agent 不再只给建议。它会获得当前节点、上游素材和对话上下文，在本地 CLI 中完成任务，并把产物写回正确的位置。

01

自动发现本机 Agent

扫描 PATH 与常用安装目录，识别已安装的 CLI、可用模型和调用协议。无需把密钥重新交给工作流。

02

带着上下文去执行

当前节点、上游文本与素材、用户指令会被整理成结构化任务，让 Agent 清楚“要做什么、参考什么、写回哪里”。

03

可靠回写工作流

运行状态、心跳、结果与错误都可追踪。增量 patch 与 revision 校验，避免并行创作时覆盖掉其他节点更新。

Agent-safe CLI

一条命令，
跑完一个节点。

rvf 面向人，也面向 Agent。默认输出 JSON，并内置节点运行的 start、heartbeat、complete 与 fail 生命周期。

自动读取上游节点 文本 Agent 与视觉模型自动分流 失败状态与产物统一回写

# Terminal A · 启动本地服务与工作流编辑器
pnpm install
pnpm dev

# Terminal B · 在仓库内启用 rvf 命令
alias rvf='pnpm --filter @red-video-flow/workflow-cli start --'

# 查看本机工作流
rvf workflow list

# 让 Codex 执行一个脚本节点
rvf workflow node run default text-1 \
  --prompt "把灵感扩写成 60 秒短剧脚本" \
  --agent-id codex

灵感沿着节点，
一步步变成画面。

每个节点既是素材，也是下一步的上下文。你可以替换 Agent、重跑单个环节，或沿着同一条链路持续迭代。

NODE / 01

Script

让文本 Agent 把灵感整理成完整脚本、台词和节奏。

NODE / 02

Storyboard

读取脚本节点，拆分镜头、景别、构图和人物动作。

NODE / 03

Image

把分镜描述送入视觉模型，生成可追踪的本地素材。

NODE / 04

Video

延续画面上下文生成视频，记录任务、文件与运行状态。

工作流在本机，
选择权也在你手里。

浏览器只负责编辑体验，本地服务管理工作流、素材与 Agent 进程。你可以清楚知道每一次调用从哪里出发、由谁执行、结果写到哪里。
